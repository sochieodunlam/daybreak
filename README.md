# Daybreak

*Take a working coffee site and make it one you'd actually ship.*

## Overview

The hardest part of product and design engineering is not making something from a clean brief. It is walking into something that already exists and turning it into something people want to use. That is most real frontend work, and the signal is in the judgment you bring.

You're handed a working marketing site for a fictional coffee subscription, Daybreak Coffee Co. The content is all there. Your job is to turn it into a site you would actually ship.

## Problem Statement

Rebuild the provided Daybreak Coffee site into a single-page marketing site you would actually ship. The content is all in the existing code; what you do with it is up to you. There is no single correct answer; this is a test of taste, craft, and judgment.

## Getting Started

### Prerequisites
- Node.js 20+
- The starter is Vite + React + TypeScript (Framer Motion is included). Use any frontend stack you are comfortable with.

### Setup
Dependencies are installed automatically when you initialize the assessment with the Litmus CLI. You're ready to start coding.

What's in the workspace:
- `src/App.tsx` is the existing site, with all of the content and copy in it. Everything you need is in here.
- `index.html`, `src/main.tsx`, `vite.config.ts`, `package.json`, `tsconfig.json` are the starter scaffolding.
- Replace and restructure as you see fit. You own the redesign.

## Requirements

1. Use the content from the existing site. This is the Daybreak Coffee site, not a different product.
2. Your build must run in the browser.
3. Submit all the source for your build.

## Notes

There is no reference solution; this is open-ended by design. The design decisions are entirely yours.

## Submission Guidelines

### What to Submit
- All source code for your redesigned site.

### How to Submit
```bash
litmus submit
```
